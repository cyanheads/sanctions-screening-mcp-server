# 0.1.5 — 2026-06-20

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.8` → `^0.10.9`** — framework-maintenance refresh; re-syncs the fresh-scaffold `scripts/` guards and `devcheck` wiring with no server source or behavior change.
- **`scripts/check-dependency-specifiers.ts`** (new, wired into `devcheck` as the `Dependency Specifiers` step) — rejects floating dependency specifiers (`latest`, `*`, and `next`/`beta`/`canary`/`rc` dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` specifier map. `bun update --latest` writes the literal `latest` dist-tag into the lock's workspace map while the concrete resolution is still correct, so it clears every other gate and lets the next `bun install` re-resolve a dep past its `package.json` range and any intentional version hold. ([mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **`scripts/lint-packaging.ts`** now validates the plugin marketplace manifests (`.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, `.codex-plugin/mcp.json`): non-empty descriptions, display fields (`name`, server key, `interface.displayName`) carrying the unscoped machine name, and the `npx -y` install arg carrying the full scoped `package.json` `name`. The `Packaging` devcheck step now runs when `manifest.json` **or** any plugin manifest is present. Opt out with `"packaging": { "pluginManifests": false }` in `devcheck.config.json`. ([mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`devcheck.config.json`** — added `"packaging": { "pluginManifests": true }` to enable the plugin-manifest checks.

### Dependency bumps
