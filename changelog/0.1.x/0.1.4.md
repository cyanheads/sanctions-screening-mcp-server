# 0.1.4 — 2026-06-19

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.8`** — adopts two framework releases:
  - `ctx.content` collector on `Context` for image/audio blocks (bytes ride `content[]` only, never `structuredContent`). ([mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239))
  - Canvas SQL gate now classifies a `SELECT`-shaped statement that parses but fails to prepare (mistyped column, unknown function) as `ValidationError` with `data.reason: 'invalid_sql'` and a `data.binderMessage`, instead of the misleading `non_select_statement`. ([mcp-ts-core#236](https://github.com/cyanheads/mcp-ts-core/issues/236))
  - `DuckdbProvider.describe({ tableName })` qualifies its pushed `WHERE` filters with the `t` alias, fixing the ambiguous-column Binder Error on every filtered `describe()`. ([mcp-ts-core#235](https://github.com/cyanheads/mcp-ts-core/issues/235))
  - `OTEL_SERVICE_NAME` is seeded from `createApp({ name })` when unset; docs corrected (no behavior change). ([mcp-ts-core#233](https://github.com/cyanheads/mcp-ts-core/issues/233))
- **`scripts/` fresh-scaffold guards re-synced** from the framework: `devcheck.ts`, `check-framework-antipatterns.ts`, and `build-changelog.ts` skip their git-dependent checks (`git grep`/`git ls-files`, changelog rollup) cleanly when `.git` is absent instead of failing with exit 128; `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree instead of crashing with ENOENT. ([mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [mcp-ts-core#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [mcp-ts-core#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
- **Agent skills re-synced** to the framework `0.10.7` baseline — propagates body drift in seven external skills that shipped at equal versions. ([mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238))
- **`devcheck.config.json`** — emptied the `outdated` allowlist (`@cyanheads/mcp-ts-core`, `@types/node`); both are now at their current resolved versions, so the entries no longer suppress a real drift signal.

### Dependency bumps

- `@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.8`
- `@types/node` `25.9.3` → `26.0.0`
